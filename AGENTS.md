# 짤 콜렉터 사양서

## 1. 이 앱의 목적

짤 콜렉터는 사용자가 웹페이지를 탐색하는 동안 조건에 맞는 이미지를 자동으로 수집하는 Chrome 확장 프로그램이다.

- 확장을 켜면 이후 방문하는 일반 웹사이트에서 이미지 수집을 수행한다.
- 실제 저장은 현재 포커스된 활성 웹 탭에서만 진행한다.
- 현재 화면에 렌더링되어 사용자가 볼 수 있는 이미지와 배경 이미지만 수집한다.
- 코드에만 연결되어 있거나, 숨김 처리되었거나, 현재 화면 밖에 있는 이미지는 수집하지 않는다.
- `img`, 실제 선택된 `srcset` 현재 소스, 보이는 CSS `background-image` 후보를 수집 대상으로 확인한다.
- 최소 너비와 높이 조건으로 저장 대상을 필터링하며, 기본 최소 크기는 `480 x 480`이다.
- 페이지 로드 후 `0.5~1`초 사이의 랜덤 대기 뒤 수집을 시작하고, 이미지는 하나씩 저장한다.
- 이미지 저장이 끝난 뒤 다음 이미지까지 `0.1~1.5`초 사이의 랜덤 대기를 둔다.
- 같은 호스트에서 다운로드 실패가 3회 연속 발생하면 해당 사이트 수집만 10분 동안 중단한다.
- 팝업 통계는 오늘 날짜 기준으로 누적하며, 페이지 이동으로 초기화하지 않는다.
- 오늘 받은 이미지 목록은 원본 파일명, 페이지 전체 주소, 이미지 전체 주소로 저장한다.
- 같은 이미지 주소는 당일 다시 다운로드하지 않는다.
- 저장 폴더는 Chrome 기본 다운로드 폴더 아래의 하위 폴더로 지정한다.
- 오늘 받은 이미지 목록은 확장 로컬 저장소에 저장한다.
- 옵션 화면에서 오늘 받은 이미지 목록을 `download-list.json` 형식으로 열 수 있다.
- 옵션 화면에서 오늘 받은 이미지 목록 뷰어를 열 수 있다.
- 기본값으로 저장 폴더 아래에 `20260621` 형식의 날짜별 폴더를 만들어 저장한다.
- 짤 콜렉터가 저장한 항목만 Chrome 다운로드 목록에서 숨길 수 있다.
- 옵션 화면에서 최근 수집, 스캔, 다운로드 로그를 확인하고 비울 수 있다.

## 2. 업데이트 방식

최근 추가된 업데이트 방식은 GitHub Releases 기반의 수동 배포 확인 방식이다.

- 업데이트 확인 대상 저장소는 `killkimno/jjalCollector`이다.
- 최신 릴리스 확인은 GitHub Releases API의 `releases/latest`를 사용한다.
- 확장 프로그램 설치 또는 시작 시 업데이트 확인을 예약하고, 최초 확인을 수행한다.
- `chrome.alarms`를 사용해 6시간 간격으로 최신 릴리스를 다시 확인한다.
- 현재 확장 버전은 `manifest.json`의 `version`을 기준으로 판단한다.
- 릴리스 태그 또는 릴리스 이름에서 `v1.2.3` 또는 `1.2.3` 형식의 버전을 추출해 비교한다.
- 최신 릴리스 버전이 현재 버전보다 높으면 업데이트 가능 상태로 저장한다.
- 업데이트 가능 상태는 `chrome.storage.local`의 `updateStatus`에 저장한다.
- 업데이트 가능 시 확장 아이콘에 빨간 알림 표시를 추가하고, 액션 타이틀에 새 버전을 표시한다.
- 팝업에는 업데이트 버튼과 최신 버전을 표시하며, 버튼 클릭 시 GitHub 릴리스 페이지를 새 탭으로 연다.
- GitHub API 요청 실패 시 기존에 저장된 업데이트 상태를 유지하되, 오류 메시지와 확인 시간을 갱신한다.

## 3. 보안 관련 처리사항

구현은 숨겨진 크롤러처럼 별도 탐색하거나 사이트를 우회하는 방식이 아니라, 사용자의 실제 브라우징 흐름 안에서 동작하는 확장 프로그램으로 유지한다.

- 크롤러 탐지 우회를 위한 헤더 위조, 세션 위장, 인증 우회, 숨은 페이지 탐색, 백그라운드 대량 요청을 구현하지 않는다.
- 수집 대상은 사용자가 현재 볼 수 있는 활성 탭의 렌더링된 이미지로 제한한다.
- 화면 밖 이미지, `display: none` 또는 `visibility: hidden` 등으로 숨겨진 이미지, DOM이나 CSS에만 존재하고 사용자에게 보이지 않는 이미지는 다운로드하지 않는다.
- 사용자가 포커스한 활성 웹 탭에서만 저장을 진행하고, 비활성 탭이나 백그라운드 탭의 이미지는 수집하지 않는다.
- 페이지 로드 직후 즉시 대량 요청하지 않고 랜덤 대기 후 하나씩 저장해 사이트 부하를 줄인다.
- 동일 이미지 URL은 당일 중복 다운로드하지 않는다.
- 같은 호스트에서 다운로드 실패가 반복되면 10분 동안 해당 사이트 수집을 중단해 비정상 반복 요청을 막는다.
- Chrome 확장 보안 정책에 맞춰 임의의 전체 경로에 직접 저장하지 않고, Chrome 기본 다운로드 폴더 아래의 지정 하위 폴더만 사용한다.
- 사용자가 설정한 최소 크기 조건과 수집 옵션을 존중한다.
